1 面试试题

public class MethodVariables {
    public static void main(String[] args) {
        int i = 1;                        // 栈中
        String str = "hello";            // 栈中地址，值在常量池中
        Integer num = 2;                // 栈中地址，值在堆中
        int[] arr = {1,2,3,4,5,6};        // 栈中地址，值在堆中
        MyData myData = new MyData();    // 栈中地址，值在堆中

        change(i, str, num, arr, myData);

        System.out.println("i = " + i);                            // 1
        System.out.println("str = " + str);                        // hello （String的不可变性）
        System.out.println("num = " + num);                        // 2    （包装类的不可变性）
        System.out.println("arr = " + Arrays.toString(arr));    // 2,2,3,4,5,6
        System.out.println("myData.a = " + myData.a);            // 11
    }

    public static void change(int i, String str, Integer num, int[] arr, MyData myData) {
        i += 1;
        str += " world";
        num += 1;
        arr[0] += 1;
        myData.a += 1;
    }


}

class MyData{
    int a = 10;
}

2 试题考点

  • 方法的传参机制
  • String、包装类等对象的不可变性

3 方法的传参机制

  1. 形参是基本数据类型

    • 传递的是数据值，即值传递

  2. 实参是引用数据类型

    • 传递的是地址，即引用传递
    • 特殊的类型：String、包装类等对象不可变性

4 试题解析

tips：

  • 基本数据类型属于值传递，方法内变量的改变，不影响实参的值。因为方法中的形参是存在在新开辟的栈帧中的局部变量表中，与原来栈帧中的局部变量无关。
  • String、包装类虽属于引用传递，但是两者具有不可变性，一旦对象发生改变，便会产生新的对象（有新的地址），原地址处的对象值不变。
  • 数组、自定义类等都是引用传递，改变都是原地址上进行改变，最终会体现到原变量上。
