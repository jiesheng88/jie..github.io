【面试-Java】i++与++i的区别
面试 Java基础 内存分配
面试 Java基础
发布日期:   2021-05-06
阅读次数:  

1 面试题描述

运行如下代码，得到的结果：

/**
 * 体会 i++ 与 ++i 在内存中的运行不同
 */
public class Iaddadd {
    public static void main(String[] args) {
        int i = 1;
        i = i ++;
        int j = i ++;
        int k = i + ++i * i++;
        System.out.println("i="+i);        // i = 4
        System.out.println("j="+j);        // j = 1
        System.out.println("k="+k);        // k = 11
    }
}

2 JVM中虚拟机栈

虚拟机栈/Java栈是由一个个栈帧构成的。（一个栈帧就是一个方法）

每个栈帧是线程私有的，包括四部分内容：局部变量表(LVT)、操作数栈(OS)、动态链接(DL)和方法返回地址(RA)。

2.1 局部变量表

局部变量表是一组变量值存储空间，用于存放方法参数和方法内部定义的局部变量。

  • 存储方法参数，以及在方法体内的局部变量；
  • 存储的数据类型包括：基本数据类型（8种）、引用类型和returnAddress类型。

2.2 操作数栈

操作数栈，主要用于保存计算过程中的中间结果，同时作为计算过程中变量临时的存储空间。

  • 往栈中写入数据或提取数据。只负责存数据和取数据，不参与计算。

3 内存运行展示

  1. int i = 1

    对应的字节码

    0 iconst_1    // 将 int 型的 1 推送至栈顶
1 istore_1    // 把栈顶的元素弹出，并赋值给局部变量表中位置为“1”的变量，此时指变量i

    i 是main方法体中的局部变量，所以将 i 放入局部变量表中。

  1. i = i ++

    = 赋值，都是将操作数栈中的值赋给左边的变量！！！！

    对应的字节码指令

     2 iload_1            // 把局部变量表中位置为“1”的变量加载到栈顶，即把 i 的值加载到栈顶
 3 iinc 1 by 1        // 将局部变量表中位置为“1”的 i 加 1，此时局部变量表中 i 的结果为 2
6 istore_1            // 把栈顶的元素弹出，并赋值给局部变量表中位置为“1”的变量。所以 i 的值又被改为了 1。

    所以这里分为三步：

    ① 先从局部变量表中读取到 i 的值到操作数栈中；

    ② i 直接在局部变量表上自增；

    ③ 将操作数栈上读取到的 i 的值赋值给局部变量表上的 i （局部变量表上 i 之前自增的结果会被覆盖）。

    1. int j = i ++;

      对应的字节码指令：

       7 iload_1            // 把局部变量表中位置为“1”的变量加载到栈顶，即把 i 的值加载到栈顶
 8 iinc 1 by 1        // 将局部变量表中位置为“1”的 i 加 1，此时结果为 2，也就是局部变量表中 i 的结果为 2。
11 istore_2            // 把栈顶的元素弹出并赋值给局部变量表中位置为“2”的 j。所以 j 是 1，但是 i 的值已经为 2。

      先处理等式右边的，然后再处理赋值=，所以这里分为三步：

      ① 先从局部变量表中读取 i 的值到操作数栈中；

      ② 局部变量表中的 i 自增，变为2；

      ③ 将操作数栈中 i 的值（即栈顶元素）赋值给局部变量表中的 j（因为计算是直接在局部变量表上进行，所以操作数栈中的i还是1）。

  1. int k = i + ++i * i++

    对应的字节码指令：

    12 iload_1        // 把局部变量表中位置为“1”的变量加载到栈顶，即把 i 的值加载到栈顶，注意 i 的值此时是 2。
13 iinc 1 by 1    // 将局部变量表中位置为“1”的 i 加 1，然后 i 就变成 3 了
16 iload_1        // 把局部变量表中位置为“1”的变量加载到栈顶，即把 i 的值加载到栈顶
17 iload_1        // 把局部变量表中位置为“1”的变量加载到栈顶，即把 i 的值加载到栈顶，所以栈中现在是 3、3、2。
18 iinc 1 by 1    // 将局部变量表中位置为“1”的 i 加 1，然后 i 就变成 4 了，注意这个 4 并未压入栈。
21 imul            // 进行乘法计算，栈中的前两个元素计算后是 9
22 iadd            // 也就是 9 + 2，结果为 11。
23 istore_3        // 把 11 从栈顶弹出，并赋值给 k，也就是局部变量表中位置为“3”的 k 的值是 11。

    这里分为七步：

    ① 先从局部变量表中读取 i 的值到操作数栈中，此时操作数栈中的结果是 2；

    ② 局部变量表中的 i 自增，变为3；

    ③ 再从局部变量表中读取 i 的值到操作数栈中，此时操作数栈中的结果是 2 3；

    ④ 再次从局部变量表中读取 i 的值到操作数栈中，此时操作数栈中的结果是 2 3 3；

    ⑤ 局部变量表中的 i 自增，变为4；

    ⑥ 操作数栈中前两个元素弹出，进行乘法运算，然后把结果3*3=9再压入栈；

    ⑦ 操作数栈中的值弹出，进行加法运算，然后把结果9+2=11在压入栈，然后把栈顶元素赋值给局部变量表中的变量k。

  1. 测试不同运算符在一起时，操作数栈入栈出栈顺序
public class Test {
    public static void main(String[] args) {
        int a = 1;
        int b = 2;
        int c = 3;
        int d = 4;

        int e = a + b * c - d / b;
        System.out.println(e);
    }
}

对应的字节码指令：

 0 iconst_1        // 当 int 取值 -1~5 时，JVM 采用 iconst 指令将常量压入栈中。
 1 istore_1        // 将操作数栈的栈顶int数值存入第1局部变量
 2 iconst_2
 3 istore_2
 4 iconst_3
 5 istore_3
 6 iconst_4
 7 istore 4

 9 iload_1        // 局部变量表处的第2个int型变量进操作数栈（即a=1入栈），此时操作数栈中结果为 1
10 iload_2        // 局部变量表处的第3个int型变量进操作数栈（即b=2入栈），此时操作数栈中结果为 1 2
11 iload_3        // 局部变量表处的第4个int型变量进操作数栈（即c=3入栈），此时操作数栈中结果为 1 2 3
12 imul            // 取出前两个元素，进行乘法运算（2*3），然后将结果6再入栈
13 iadd            // 取出前两个元素，进行加法运算（6+1），然后将结果7入栈
14 iload 4        // 局部变量表处的第5个int型变量进操作数栈（即d=4入栈），此时操作数栈中结果为 7 4
16 iload_2        // 局部变量表处的第3个int型变量进操作数栈（即b=2入栈），此时操作数栈中结果为 7 4 2
17 idiv            // 除法
18 isub            // 减法
19 istore 5        // 栈顶元素保存到局部变量表

29 return

4 小结

  • 赋值=，最后计算
  • = 右边只要遇到变量，就会从局部变量表中将变量的值加载到操作数栈中
  • = 右边，如果遇到的都是同一级的运算符，那么从左到右依次加载变量值，并压入操作数栈
  • = 右边，如果遇到的不是同一级运算符，那么从左到右依次加载变量值，并压入操作数栈，直到遇到优先级高的，那就先计算，计算结果入栈，然后再依次加载变量值，并压入操作数栈
  • 自增、自减都是直接在局部变量表中修改变量的值，不经过操作数栈
  • 最后的赋值之前，临时结果也是存储在操作数栈中的
